# Rendering pipeline

Rendering in Fission is explicit by stage. The frame path is:

1. widget build
2. internal node graph lowering
3. layout
4. paint command generation
5. backend render (today GPU-based output paths)

## Why the staged pipeline helps

Each stage has one responsibility. That reduces the “everything changed” debugging problem and gives a precise point of measurement.

## Debugging order

- first confirm state and build output,
- then inspect constraints and layout metrics,
- then paint ordering,
- and finally render target output.

Move one stage at a time, and keep assumptions documented so regressions are easier to isolate.
