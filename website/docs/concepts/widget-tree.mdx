# Widget tree model

The widget tree is a typed description of the UI for one frame. For a fixed state input, the tree should be stable and understandable.

## Why the tree is useful

A stable tree gives you a direct line from behavior to output:

- if state is unexpected, inspect the action path first,
- if structure is unexpected, inspect branch selection in build,
- if visuals are wrong, inspect layout and paint boundaries.

## Principles

Keep build functions small and intentional:

- use one top-level composition per concern,
- keep state reads read-only,
- avoid side effects during build,
- keep each widget focused.

When this pattern is consistent, teams can reason about changes quickly and avoid accidental complexity.
