# What is Fission

Fission is aimed at teams that need a dependable UI model for multi-platform products. The value is not a bigger widget catalog; it is consistency across the entire engineering path:

- a single runtime model for desktop, web, and mobile targets,
- deterministic state transitions for predictable behavior,
- explicit handling of async work, and
- an iteration loop that starts fast.

## Why this matters

A common product pain point is architectural drift: app behavior looks good on one target but changes unpredictably on another, or “a new feature” adds side effects in UI code that are hard to trace.

Fission’s model reduces that drift by aligning:

- **UI definition** through a declarative widget tree,
- **State updates** through typed actions and reducers,
- **Side effects** through commands/services/jobs,
- **Rendering** through a fixed pipeline and predictable diagnostics.

## What Fission is explicitly for

Fission is useful when you need:

- a single source-of-truth app model,
- reusable behavior patterns across targets,
- a clear contract for async work,
- and repeatable iteration for teams.
