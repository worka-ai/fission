# What is Fission

Fission is a Rust-first UI framework and command-line workflow for building one typed app model and running it across multiple runtimes.  
You describe state and actions in Rust, then let Fission render the same widget tree through platform hosts that are added over time.

The default scaffold is intentionally desktop-first. `fission init my-fission-app` creates a runnable starter with `src/app.rs`, `src/main.rs`, `src/lib.rs`, `Cargo.toml`, `fission.toml`, and desktop target readmes.  
That starter is designed so new users can run a real app quickly and focus on behavior before platform expansion.

From install through first app, the sequence is simple: `fission init`, `cd my-fission-app`, then `cargo run`.  
That command uses the existing desktop targets (`linux`, `macos`, `windows`) and gives you an immediate visual check like the generated counter scaffold (`Count: 0`).

Use staging after your first loop is stable: expand from desktop only when the interaction path is predictable, then add extra targets with `cargo fission add-target`:

- `cargo fission add-target web`
- `cargo fission add-target ios`
- `cargo fission add-target android`

Each `add-target` call updates `fission.toml` and writes target-specific files under `platforms/<target>/` (for example `platforms/web/run-browser.sh`).  
That keeps onboarding boring where it should be boring, and only introduces platform-specific prerequisites when the team is ready for them.

Use/Don't use guidance:

- **Use Fission** when you want a single typed interaction model and a predictable path for adding web/mobile targets from one Rust source of truth.
- **Don't use Fission** when you need full native parity for web/mobile from day one, or when your architecture requires platform-specific UI logic before your state model is stable.
