# Why Fission

The best way to evaluate these differentiators is against the first generated app: if each one helps you ship and debug that tiny counter confidently, it will scale with the same discipline to the rest of your product.

- **Deterministic action flow**
  - In your first app, the increment path should produce one obvious state transition, so `Count` changes match the reducer and are easy to reproduce in tests.
  - Tradeoff: you lose "quick-and-dirty" state mutation style; all behavior must go through explicit handlers, which can feel strict at the beginning.

- **Desktop-first onboarding before platform spread**
  - In your first app, `cargo run` from the scaffold root is the first stable loop and validates the default model before you add any web/mobile target.
  - Tradeoff: additional targets are not ready on day one; each one (`web`, `ios`, `android`) needs a dedicated `cargo fission add-target` step plus host prerequisites.

- **Separation of interaction model from platform shell**
- In your first app, `src/app.rs` stays focused on state and widgets, while platform differences are handled by generated shell code under `platforms/<target>`.
- Tradeoff: platform-specific issues remain possible (for example, iOS simulator rendering can run with software fallback depending on backend capabilities), so you still need per-target verification.

- **Built-in quality pressure from day one**
  - In your first app, accessible labels, readable text, and focus order can be checked early, then carried forward when you add localization and deeper tests.
  - Tradeoff: the starter provides the shape, but not a complete localization/accessibility finish; you own the translation keys and any native bridge parity gaps.

- **Fast local confidence, then scoped expansion**
  - In your first app, you can iterate quickly (`edit`, `cargo run`, `confirm behavior`) before deciding which host to generate and stabilize next.
  - Tradeoff: desktop confidence does not automatically equal web/mobile confidence; each host still needs its own smoke check and environment setup.
