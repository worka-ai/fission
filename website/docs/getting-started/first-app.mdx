# Build your first app

The goal is to show the complete signal path: state, action, and widget rebuild.

## The core pattern

- state is immutable during build,
- actions describe user intent,
- reducers update state deterministically,
- UI reads state and dispatches intent.

```rust
use fission::prelude::*;

#[derive(Default, Debug, Clone, PartialEq, Serialize, Deserialize)]
struct CounterState {
    count: i32,
}

impl AppState for CounterState {}

#[derive(Action, Serialize, Deserialize, Debug, Clone, PartialEq, Eq)]
struct Increment;

#[derive(Action, Serialize, Deserialize, Debug, Clone, PartialEq, Eq)]
struct Decrement;

struct MyApp;

impl Widget<CounterState> for MyApp {
    fn build(&self, ctx: &mut BuildCtx<CounterState>, view: &View<CounterState>) -> Node {
        Column {
            children: vec![
                Text {
                    content: TextContent::Literal(format!("Count: {}", view.state.count)),
                    font_size: Some(24.0),
                    ..Default::default()
                }
                .into(),

                Button {
                    on_press: Some(ctx.bind(Increment, |state, _, _| {
                        state.count += 1;
                    })),
                    child: Some(Box::new(Text::new("Increment").into())),
                    ..Default::default()
                }
                .into(),

                Button {
                    on_press: Some(ctx.bind(Decrement, |state, _, _| {
                        state.count -= 1;
                    })),
                    child: Some(Box::new(Text::new("Decrement").into())),
                    ..Default::default()
                }
                .into(),
            ],
            ..Default::default()
        }
        .into()
    }
}

fn main() -> anyhow::Result<()> {
    DesktopApp::new(MyApp).run()
}
```

## Run the app

```bash
cargo run --example counter
```

## What to try next

- extract `Text` and `Button` groups into reusable widgets,
- guard updates (`count` cannot go negative),
- add a reducer function for reset and undo paths,
- move side effects into commands before adding persistence.
