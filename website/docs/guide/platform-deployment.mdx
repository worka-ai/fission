# Platform and deployment

Start with desktop for fast iteration, then expand with the same app model to web and mobile targets.

## Target matrix

- Desktop (`DesktopApp`) today.
- Web/WASM via browser host.
- Native iOS and Android flows through CLI-generated projects.

## Typical build flow

1. `fission init my-app`
2. implement core app and reference-state transitions,
3. add platform configuration,
4. run desktop smoke,
5. run `web` target to validate UI behavior in browser,
6. package native targets.

## Deployment note

Use environment variables and command/service hooks for platform-specific settings. Keep secrets out of app sources and pass credentials through CI tooling.
