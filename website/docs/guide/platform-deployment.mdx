# Platform and deployment

Keep behavior in `src/app.rs` and use generated targets for runtime-specific run commands.

After `fission init`, generated project output includes:

- `src/main.rs`, `src/lib.rs`, `src/app.rs`
- `Cargo.toml`
- `assets/app-icon.png`
- `fission.toml`
- `README.md` and `platforms/<target>/README.md`

`fission.toml` is the target manifest (`linux`, `macos`, `windows` by default and added targets as you scaffold more).

Use `.with_sync_env(|state, env| { ... })` in your app entry when locale/theme/test settings live in state.

```rust
DesktopApp::new(HomeApp)
    .with_env(base_env())
    .with_sync_env(|state, env| {
        env.locale = state.locale.clone();
    })
    .run()
```

## Base local flow (generated app)

```bash
fission init my-app
cd my-app
cargo run
```

Expected output:

- desktop window opens with the initial UI from `src/app.rs`
- state-driven interactions are immediately usable

Next step:

- add targets once local behavior is stable:

```bash
cargo fission add-target web ios android
```

## Platform-specific run commands

<Tabs>
<TabItem value="web" label="Web (WASM)" default>

Prerequisites:

- `rustup target add wasm32-unknown-unknown`
- `cargo install wasm-pack`

Run:

```bash
./platforms/web/run-browser.sh
```

Expected output:

- builds the wasm package (`wasm-pack`)
- prints a local URL like `Serving http://127.0.0.1:8123/platforms/web/`
- starts a local server waiting for `Ctrl+C`

Next step:

- open the printed URL and verify UI behavior.

</TabItem>
<TabItem value="ios" label="iOS (simulator)">

Prerequisites:

- `rustup target add aarch64-apple-ios aarch64-apple-ios-sim`
- install Xcode and verify simulator SDK:

```bash
xcrun --sdk iphonesimulator --show-sdk-path
```

Run:

```bash
./platforms/ios/run-sim.sh
```

Expected output:

- packages the simulator app bundle
- boots/uses an available iPhone simulator
- launches the app in that simulator

Notes:

- if the target requires environment sync, pass it before launch:

```bash
FISSION_TEST_CONTROL_PORT=48761 ./platforms/ios/run-sim.sh
```

</TabItem>
<TabItem value="android" label="Android (emulator)">

Prerequisites:

- `rustup target add aarch64-linux-android`
- Android SDK + NDK and env:
  - `ANDROID_HOME`
  - `ANDROID_NDK`

Run:

```bash
./platforms/android/run-emulator.sh
```

Expected output:

- creates/uses `FissionApi32Arm64` emulator if needed
- builds and signs APK
- installs APK and prints `APK=<path>`

Notes:

- optional test-control bridge:

```bash
FISSION_TEST_CONTROL_PORT=48761 ./platforms/android/run-emulator.sh
```

</TabItem>
</Tabs>

## Command matrix (generated app)

Desktop path:

- `cargo run`

Web path:

- `./platforms/web/run-browser.sh`

iOS path:

- `./platforms/ios/run-sim.sh`

Android path:

- `./platforms/android/run-emulator.sh`

Generated files to check before release:

- `fission.toml`

- `platforms/android/AndroidManifest.xml`, `platforms/android/package-apk.sh`, `platforms/android/run-emulator.sh`
