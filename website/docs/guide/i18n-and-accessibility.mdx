# i18n and accessibility

Product quality depends on reaching more users. Fission treats localization and accessibility as part of baseline app quality.

## i18n first

- Keep user-facing text keys and localizable resources grouped by feature.
- Avoid hard-coded strings in rendering logic.
- Validate overflow and RTL behavior early.

## Accessibility baseline

- semantic labels and roles,
- keyboard focus order,
- contrast and visible focus,
- screen-reader-friendly content hierarchy.

## Checklist

- Are every interactive control labelable?
- Can each screen be navigated without pointer?
- Is copy localizable without logic changes?
- Do color and spacing pass accessibility checks?

Use this as part of every milestone, not a last-pass polish task.
