# i18n and accessibility

Many teams add localization and accessibility after first build. Fission works best when these are included from the start.

Use externalized text keys for all user-facing copy. Keep labels and roles explicit for controls.

Use this checklist before marking a screen complete:

- every interactive control has a clear label,
- focus order matches the intended navigation,
- focus indicators are visible,
- text passes readability checks for longer strings,
- keyboard interaction works on non-pointer environments.

If this is your first implementation pass, add at least one a11y and one i18n check before expanding to the next feature.

Use it with [Platform and deployment](/docs/guide/platform-deployment) because these checks matter across targets.
