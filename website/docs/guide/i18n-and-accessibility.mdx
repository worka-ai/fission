# i18n and accessibility

Localization and accessibility are quality gates, not optional enhancements.

## i18n defaults

Use stable keys, central resources, and keep copy out of build expressions where possible.

## Accessibility defaults

Every interactive element should have a clear label and predictable focus behavior. Keyboard navigation should match layout intent.

## Practical checklist

- semantic labels for buttons, fields, and landmarks,
- visible focus and predictable tab order,
- contrast checks and spacing for reading comfort,
- localization tests for longer or right-to-left text.

Make these checks part of your normal development cadence.
