# Playground-first workflow

A fast feedback loop is a multiplier. We treat it as the first workflow after setup.

## Suggested rhythm

1. Open an app example.
2. Edit one behavior at a time (state or layout).
3. Run and validate interaction in browser.
4. Copy the final state into your local project.
5. Verify edge cases and tests before packaging.

## What you get in exchange

- immediate visual confirmation,
- faster confidence on layout and spacing,
- quicker debugging with isolated examples.

## Useful checkpoints

- Keep an example branch for playground experiments.
- Move stable logic into the real app once interactions are validated.
- Add one regression test per behavior before broadening target matrix.
