# Playground-first workflow

The best loop is quick and small:

1. change one behavior,
2. validate in browser preview,
3. copy stable behavior into local project,
4. add tests around the validated behavior,
5. expand to next behavior.

The loop is useful even for complex projects because it isolates variables; you are validating UI and behavior before platform-specific packaging overhead enters.

Keep a dedicated playground branch for experiments and move only stabilized changes into main app code.
