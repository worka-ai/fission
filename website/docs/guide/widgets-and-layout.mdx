# Widgets and layout

The most maintainable screens start as small composable widgets.

## Build a screen in layers

1. identify container structure,
2. build simple child widgets,
3. encode alignment and spacing explicitly,
4. keep text and control behavior in domain components.

```rust
Root {
  child: Some(Box::new(
    Column {
      children: vec![
        Header::new("Dashboard").into(),
        Body::new(...).into(),
        Footer::new("Status").into(),
      ],
    }
    .into(),
  )),
  ..Default::default()
}
```

## Why this style scales

Because layout complexity is often the first source of platform divergence. Explicit structure and constraints make it easier to tune wide-screen behavior without rewriting logic.
