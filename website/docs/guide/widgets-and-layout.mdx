# Widgets and layout

A screen is easier to reason about when it is layered from outside to inside.

Start with containers (`Column`, `Row`, `Grid`, `Scroll`, `SafeArea`, `ZStack`) and make
structure explicit before adding visual detail.

```rust
Column {
    children: vec![
        Text::new("Dashboard").into(),
        Row {
            children: vec![
                Text::new("Left rail").into(),
                Text::new("Main body").into(),
            ],
            ..Default::default()
        }.into(),
        Text::new("Status").into(),
    ],
    ..Default::default()
}
```

If your screen starts to feel deeply nested, split structure into named widgets before you
add behavior. This keeps wide-screen behavior stable because spacing and bounds remain visible
in each component.

After layout patterns are comfortable, move to [State and actions](/docs/guide/state-and-actions).
