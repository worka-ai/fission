# Commands, services, and jobs

Use this split whenever behavior reaches outside pure build/update logic.

- **Command**: one-shot request with bounded output.
- **Service**: shared, reusable integration object.
- **Job**: repeating, long-lived, or cancellable lifecycle behavior.

```text
UI action -> command request -> service/job -> result action -> reducer
```

## Selection guide

- Need one response then done? start with **command**.
- Need persistent behavior (cache, API client, storage)? use a **service**.
- Need scheduled/cancellable recurring work? use a **job**.

This separation helps with testing and platform behavior consistency.
