# Start here

If you are evaluating Fission for a product that must ship on more than one platform, this is the place to begin.

You do not need to read everything at once. The practical path is:

1. understand the architecture choice in **What is Fission**,
2. create a working project in **Install and setup**,
3. build a baseline feature in **Build your first app**,
4. continue with the narrative tutorials and guides before consulting reference pages.

That order is intentional. It lets you decide quickly whether the model fits your team before deep-dive API lookup.

When you are ready, open:

- [What is Fission](/docs/getting-started/what-is-fission)
- [Install and setup](/docs/getting-started/install)
- [Build your first app](/docs/getting-started/first-app)

Need fast verification? Use the [playground workflow](/docs/guide/playground-driven-workflow) after your first app so UI and behavior can be validated without waiting on full target builds.
