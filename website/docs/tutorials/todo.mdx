# Todo application tutorial

A todo flow introduces three useful concepts at once: list state, action-driven updates, and background work.

```rust
fn hydrate(state: &mut TodoState, action: LoadTodos, _ctx: &mut ReducerContext<TodoState>) {
    state.loading = true;
}
```

Use typed actions for local updates (`TodoAdded`, `TodoToggled`, `TodosLoaded`) and use a command/service boundary for persistence.

A good test order is:

- state transitions,
- command lifecycle,
- UI interaction validation.

If you are learning, do not add database and sync details until this loop is stable.
