# Todo application tutorial

Use this tutorial to combine local state and async work.

## Data model

- list state,
- action types for add/toggle/load,
- command lifecycle for persistence.

## Suggested action set

- `TodoAdded { title: String }`
- `TodoToggled(usize)`
- `TodosLoaded(Vec<Todo>)`

## Why commands matter

Store interaction state locally and move storage/network effects out of reducer functions. Commands keep the tree and UI logic deterministic.

```rust
fn hydrate(state: &mut TodoState, action: LoadTodos, _ctx: &mut ReducerContext<TodoState>) {
    state.loading = true;
}
```

When hydrated, the completion action updates local state and UI renders from state.
