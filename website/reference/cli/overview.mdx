# CLI overview

The CLI currently does two important things:

1. `fission init <path>` creates a runnable app scaffold.
2. `cargo fission add-target <targets...>` adds host folders to an existing app.

## `fission init`

Important flags:

- `--name` to override the crate/package name
- `--app-id` to control the mobile app identifier
- `--local-path` to point the scaffold at a local Fission checkout

## `cargo fission add-target`

Important flag:

- `--project-dir` to target an existing scaffold outside the current directory

## Generated files worth knowing

- `Cargo.toml`
- `src/main.rs`
- `src/lib.rs`
- `src/app.rs`
- `fission.toml`
- `platforms/<target>/README.md`
- `assets/app-icon.png`

## Current scope

The CLI is intentionally narrow: it gets a shared app skeleton on disk and gives you concrete host paths for desktop, web, iOS, and Android.

For target reality and prerequisites, pair this page with [Targets](/reference/platform/targets).
