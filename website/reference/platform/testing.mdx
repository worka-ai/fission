# Testing

This page focuses on the shell side of testing rather than the headless runtime helpers.

## Test control server

Desktop and mobile shells can expose an HTTP test control server through `with_test_control_port(...)` or the `FISSION_TEST_CONTROL_PORT` environment variable.

That server is what `LiveTestClient` talks to.

## What the live driver can ask for

- tap, drag, scroll, and resize
- keyboard and text input
- screenshots
- visible text
- semantic tree
- pump and wait commands

## Smoke-path coverage in the repo

The repo uses live-driver tests across counter, widget gallery, text lab, animation gallery, chart gallery, inbox, terminal, and editor examples.

Mobile and web also have dedicated smoke scripts under `examples/mobile-smoke/` and `examples/web-smoke/`.
