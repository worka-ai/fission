# Testing

Testing is strongest when it matches your app architecture.

Use reducer unit tests first, then command/service tests, then UI flow checks and target smoke checks.

Capture failures by stage:

- action and reducer mismatch,
- command lifecycle mismatch,
- job start/stop mismatch,
- layout or rendering regressions.

This gives repeatable debugging across desktop and web targets.
