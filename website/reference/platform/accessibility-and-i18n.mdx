# Accessibility and i18n

This section defines baseline requirements for accessible and localized applications.

## Accessibility

- semantic labels and roles,
- keyboard and focus order,
- visible focus states,
- a11y-friendly structure for dynamic content.

## i18n

- key-based string management,
- locale-specific resource bundles,
- tests for long strings and layout shifts,
- direction-aware layout checks where applicable.
