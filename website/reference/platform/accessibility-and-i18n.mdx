# Accessibility and i18n

This section is your checklist when preparing your first release candidate.

For accessibility, verify labels, role semantics, focus order, and visible keyboard state.

For internationalization, keep user-facing text keyed, test long strings, and check layout stability for direction-sensitive text.

Applying these checks in the first iteration avoids large refactors once feature count grows.
