# Targets

Fission keeps product logic in shared Rust code and uses target folders as host wrappers.

## Generated target names

The CLI can scaffold:

- `windows`
- `macos`
- `linux`
- `web`
- `ios`
- `android`

## Commands you will actually run

- desktop: `cargo run`
- web: `./platforms/web/run-browser.sh`
- iOS: `./platforms/ios/run-sim.sh`
- Android: `./platforms/android/run-emulator.sh`

## Repo-backed smoke references

- `examples/web-smoke/`
- `examples/mobile-smoke/`

Those checked-in examples are the source of truth for what the generated host paths should look like.

## What each target folder owns

- packaging and host metadata
- launcher scripts and prerequisites
- platform-specific README notes
- app icon and bundle wiring

It should not own your product logic.
