# Rendering pipeline

The frame follows four major phases after a build result is produced: graph lowering, layout, paint command generation, and render backend output.

Keep this map in mind for performance and correctness work:

- bad geometry usually belongs to layout,
- wrong stacking usually belongs to paint,
- unstable behavior usually belongs to state/actions.

Use the pipeline to localize changes quickly during iteration.

For deeper behavior checks, combine this with testing and observability guidance.
