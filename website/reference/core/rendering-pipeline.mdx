# Rendering pipeline

A single rendering pass progresses through staged boundaries:

1. widget build,
2. internal graph lowering,
3. layout,
4. paint command creation,
5. render backend output.

## Why this matters

Stable pipelines make timing and regression analysis easier. You can isolate cost and bugs by stage and keep performance work actionable.

## Diagnostics pairing

Use with testing/observability notes when behavior differs by target or interaction sequence.
