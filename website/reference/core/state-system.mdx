# State system

The state system is the backbone of Fission behavior.

## Core concepts

- `AppState`: typed state root used for snapshotting and rebuilds,
- `Action`: typed intent that enters mutation flow,
- `Reducer`: deterministic mutation function,
- `ReducerContext`: runtime metadata and helper APIs.

## Flow

1. action arrives,
2. reducer receives `&mut state`,
3. reducer updates state,
4. next build derives output.

## Best practices

- keep state serializable when practical,
- isolate small reducer concerns,
- avoid implicit updates outside action flow,
- test transitions in unit tests before expanding integration paths.
