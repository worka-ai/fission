# Commands, services, and jobs

This page is a quick lookup for async behavior.

A command is one-shot and bounded. A service is reusable across the app. A job is lifecycle-aware and can run repeatedly or be cancelled.

```text
action -> command/service/job -> completion action -> reducer -> render
```

Use the split to prevent side effects from entering build logic.

When debugging production issues, start by tracing this sequence to identify whether work was triggered by user input or timed lifecycle behavior.
