# Commands, services, and jobs

These primitives define behavior outside synchronous render.

## Command

A command is a bounded async request and response.

## Service

A service is a reusable, long-lived integration abstraction.

## Job

A job is lifecycle-aware background work with scheduling/cancellation semantics.

## Recommended structure

```text
action -> command/service/job -> completion action -> reducer
```

This keeps async behavior auditable and testable.
