# Widget trait

Every widget in the UI implements the same build contract.

```rust
fn build(&self, ctx: &mut BuildCtx<S>, view: &View<S>) -> Node
```

`S` is your application state type.

Use this contract as a rule:

- `build` reads state from `view`,
- binds user interactions through `ctx`,
- returns a `Node` that represents the next render output.

Keep `build` deterministic. If rendering logic changes based on side effects, behavior becomes difficult to test and replay.
