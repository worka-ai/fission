# Widget trait

Every renderable unit in Fission implements the same contract:

```rust
fn build(&self, ctx: &mut BuildCtx<S>, view: &View<S>) -> Node
```

## What belongs in `build()`

- read current state from `view.state`,
- read runtime and environment data from `view.runtime` and `view.env`,
- bind actions, declare resources, and register overlays through `ctx`,
- return a deterministic `Node` tree.

## What does not belong in `build()`

- direct I/O,
- hidden mutation outside reducers,
- platform API calls,
- long-lived work that is not represented as a resource or effect.

## Important supporting types

- `View<S>`: read-only state, runtime state, environment, and prior layout access
- `BuildCtx<S>`: action binding plus runtime declarations
- `Node`: the authoring tree that gets lowered into Core IR
- `WidgetNodeId`: stable IDs for runtime-owned behavior like animation and overlays

## Common implementation patterns

- simple screens: implement `Widget<S>` directly on a struct
- reusable helpers: return `Node` or implement `Widget<S>` on a small view-model wrapper
- custom surfaces: return `Node::Custom(...)` with a `LowerDyn` lowerer
