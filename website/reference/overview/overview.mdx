# Reference overview

The reference is organized by subsystem first and widgets second.

## Core runtime

Use the core section when you need the shared contracts that every shell and app depend on:

- `Widget`, `View`, and `BuildCtx`
- actions, reducers, selectors, and effect routing
- resources, capabilities, jobs, services, and timers
- environment sync, input, text-edit state, IME, clipboard, and runtime-owned media state
- animation, portals, embed registration, and shell integration

Start with:

- [Widget trait](/reference/core/widget-trait)
- [State system](/reference/core/state-system)
- [Resources and capabilities](/reference/core/resources-and-capabilities)

## Shells, targets, and tooling

Use the shell and tooling pages when you need the host boundary:

- desktop, mobile, and web builder APIs,
- current target and smoke-path reality,
- live test control,
- diagnostics,
- CLI-generated project scaffolds.

Start with:

- [Platform runtime](/reference/core/platform-runtime)
- [Targets](/reference/platform/targets)
- [CLI overview](/reference/cli/overview)

## Widgets

The widget pages are still complete, but they are grouped by job:

- layout and structure,
- forms and input,
- navigation and overlays,
- display and feedback,
- media, embeds, and escape hatches.

Start with [Widget catalog](/reference/widgets/catalog).
