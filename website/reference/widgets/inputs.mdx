# Input widgets

Input components are where user intent enters your model.

## Core expectations

- events dispatch typed actions,
- handlers should remain minimal,
- validation belongs to state transitions,
- async side effects should flow through command pathways.

## Pattern

`UI event -> action -> reducer` is default; escalate to `command/service/job` when work leaves the synchronous frame.
