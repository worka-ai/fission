# Input widgets

Inputs map to explicit actions first. Keep handlers small, then move async work to commands,
services, or jobs.

Core input widgets:
`Button`, `Checkbox`, `Radio`, `Slider`, `Switch`, `TextInput`, `NumberInput`, `Select`,
`SelectItem`, `Menu`, `MenuButton`, `MenuItem`, `Combobox`, `RangeSlider`, `DropDown`,
`DatePicker`, `DateRangePicker`, `TimePicker`, `FileUpload`, and `Editable`.

Pattern:

- user interaction -> action,
- action -> reducer or async boundary,
- async response -> completion action -> state update.

If a value comes from validation or persistence, treat that as command/service responsibility.
