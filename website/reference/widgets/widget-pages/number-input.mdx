# NumberInput

An input-focused control for converting user intent into typed actions.


## What this is for
`NumberInput` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::NumberInput;

let node = NumberInput {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `Option<WidgetNodeId>`
- `value`: `f32`
- `display_text`: `Option<String>`
- `min`: `Option<f32>`
- `max`: `Option<f32>`
- `step`: `f32`
- `field_width`: `Option<f32>`

- `gap`: `Option<f32>`
- `on_increment`: `Option<ActionEnvelope>`
- `on_decrement`: `Option<ActionEnvelope>`
- `on_change`: `Option<ActionEnvelope>`
