# Menu

An input-focused control for converting user intent into typed actions.


## What this is for
`Menu` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::Menu;

let node = Menu {
    ..Default::default()
}.into_node();
```

## Config surface
- `items`: `Vec<MenuItem>`
- `width`: `Option<f32>`
- `max_height`: `Option<f32>`
