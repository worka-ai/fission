# Transition

A public widget/API entrypoint in the Fission surface.


## What this is for
`Transition` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::Transition;

let node = Transition {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `WidgetNodeId`
- `value`: `f32`
- `property`: `AnimationPropertyId`
- `duration`: `u64`
- `delay`: `u64`
- `child`: `Box<Node>`
