# TreeItem

A utility or bridge type used while composing higher-level widget hierarchies.


## What this is for
`TreeItem` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::TreeItem;

let node = TreeItem {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `String`
- `icon`: `Option<String>`
- `label`: `String`
- `children`: `Vec<TreeItem>`
- `on_toggle`: `Option<ActionEnvelope>`
- `on_select`: `Option<ActionEnvelope>`
