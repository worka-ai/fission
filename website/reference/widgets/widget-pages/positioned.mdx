# Positioned

A layout and composition primitive for predictable structure and spacing.


## What this is for
`Positioned` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_core::ui::Positioned;

let node = Positioned {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `Option<NodeId>`
- `left`: `Option<f32>`
- `top`: `Option<f32>`
- `right`: `Option<f32>`
- `bottom`: `Option<f32>`
- `width`: `Option<f32>`
- `height`: `Option<f32>`
- `child`: `Option<Box<Node>>`
