# Button

An input-focused control for converting user intent into typed actions.


## What this is for
`Button` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_core::ui::Button;

let node = Button {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `Option<NodeId>`
- `child`: `Option<Box<Node>>`
- `on_press`: `Option<ActionEnvelope>`
- `semantics`: `Option<Semantics>`
- `width`: `Option<f32>`
- `height`: `Option<f32>`
- `min_width`: `Option<f32>`
- `max_width`: `Option<f32>`
- `flex_grow`: `f32`
- `flex_shrink`: `f32`
- `padding`: `Option<[f32; 4]>`
- `style`: `Option<ButtonStyleOverride>`
