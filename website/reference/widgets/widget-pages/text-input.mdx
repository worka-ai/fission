# TextInput

An input-focused control for converting user intent into typed actions.


## What this is for
`TextInput` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_core::ui::TextInput;

let node = TextInput {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `Option<NodeId>`
- `value`: `String`
- `label`: `Option<TextContent>`
- `placeholder`: `Option<TextContent>`
- `helper_text`: `Option<TextContent>`
- `error_text`: `Option<TextContent>`
- `counter_text`: `Option<TextContent>`
- `on_change`: `Option<ActionEnvelope>`
- `on_submit`: `Option<ActionEnvelope>`
- `on_editing_complete`: `Option<ActionEnvelope>`
- `on_tap_outside`: `Option<ActionEnvelope>`
- `width`: `Option<f32>`
