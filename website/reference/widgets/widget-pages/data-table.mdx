# DataTable

A product-surface widget for status, content grouping, and communication.


## What this is for
`DataTable` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::DataTable;

let node = DataTable {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `WidgetNodeId`
- `columns`: `Vec<TableColumn>`
- `rows`: `Vec<TableRow>`
- `selected_ids`: `Vec<String>`
- `on_selection_change`: `Option<Arc<dyn Fn(String) -> ActionEnvelope + Send + Sync>>`
