# CircularProgress

A product-surface widget for status, content grouping, and communication.


## What this is for
`CircularProgress` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::CircularProgress;

let node = CircularProgress {
    ..Default::default()
}.into_node();
```

## Config surface
- `value`: `Option<f32>`
- `size`: `f32`
- `color`: `Option<Color>`
- `track_color`: `Option<Color>`
- `thickness`: `f32`
