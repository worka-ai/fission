# Calendar

A product-surface widget for status, content grouping, and communication.


## What this is for
`Calendar` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::Calendar;

let node = Calendar {
    ..Default::default()
}.into_node();
```

## Config surface
- `year`: `i32`
- `month`: `u32`
- `selected_date`: `Option<NaiveDate>`
- `on_select`: `Option<Arc<dyn Fn(NaiveDate) -> ActionEnvelope + Send + Sync>>`
- `on_navigate`: `Option<Arc<dyn Fn(i32`
- `cell_size`: `Option<f32>`
- `padding`: `Option<f32>`
