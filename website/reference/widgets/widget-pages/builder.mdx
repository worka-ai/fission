# Builder

A utility or bridge type used while composing higher-level widget hierarchies.


## What this is for
A closure-based widget that builds a [`Node`] tree from a function. `Builder` lets you define inline widgets without creating a named struct.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_core::ui::Builder;

let node = Builder {
    ..Default::default()
}.into_node();
```

## Config surface
- Use `Default::default()` to start and set only required properties.
- Prefer small, explicit updates so diffs stay readable.
