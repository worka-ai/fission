# DropDown

An input-focused control for converting user intent into typed actions.


## What this is for
`DropDown` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::DropDown;

let node = DropDown {
    ..Default::default()
}.into_node();
```

## Config surface
- `on_toggle`: `Option<ActionEnvelope>`
- `options`: `Vec<String>`
- `on_select`: `Option<ActionEnvelope>`
- `selected`: `Option<String>`
