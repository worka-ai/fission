# TimePicker

An input-focused control for converting user intent into typed actions.


## What this is for
`TimePicker` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::TimePicker;

let node = TimePicker {
    ..Default::default()
}.into_node();
```

## Config surface
- `hour`: `u32`
- `minute`: `u32`
- `on_change`: `Option<Arc<dyn Fn(u32`
