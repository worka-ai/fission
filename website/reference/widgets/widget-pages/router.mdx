# Router

A navigation utility used to shape app-level flow and compositional patterns.


## What this is for
`Router` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::Router;

let node = Router {
    ..Default::default()
}.into_node();
```

## Config surface
- `current_path`: `String`
- `routes`: `Vec<Route<S>>`
- `not_found`: `Option<PageBuilder<S>>`
