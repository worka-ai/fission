# LayoutBuilder

A public widget/API entrypoint in the Fission surface.


## What this is for
A closure-based widget that receives its parent's [`BoxConstraints`]. `LayoutBuilder` is the layout-aware counterpart of [`Builder`]. The closure receives the constraints from the previous frame (or a fallback derived from the viewport) so it can adapt its output to the available space.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_core::ui::LayoutBuilder;

let node = LayoutBuilder {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `Option<WidgetNodeId>`
- `flex_grow`: `f32`
- `flex_shrink`: `f32`
