# Scroll

A layout and composition primitive for predictable structure and spacing.


## What this is for
`Scroll` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_core::ui::Scroll;

let node = Scroll {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `Option<NodeId>`
- `child`: `Option<Box<Node>>`
- `direction`: `FlexDirection`
- `width`: `Option<f32>`
- `height`: `Option<f32>`
- `show_scrollbar`: `bool`
- `flex_grow`: `f32`
- `flex_shrink`: `f32`
