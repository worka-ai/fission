# DatePicker

An input-focused control for converting user intent into typed actions.


## What this is for
`DatePicker` is part of Fission's widget API and fits the deterministic, state-driven render model.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::DatePicker;

let node = DatePicker {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `WidgetNodeId`
- `value`: `Option<NaiveDate>`
- `is_open`: `bool`
- `width`: `Option<f32>`
- `on_change`: `Option<Arc<dyn Fn(NaiveDate) -> ActionEnvelope + Send + Sync>>`
- `on_toggle`: `Option<ActionEnvelope>`
- `on_close`: `Option<ActionEnvelope>`
