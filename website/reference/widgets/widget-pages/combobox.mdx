# Combobox

An input-focused control for converting user intent into typed actions.


## What this is for
A searchable dropdown that combines a text input with a filterable item list. The user types into a `TextInput` (the trigger), which filters the available items. The filtered list is displayed in a [`Popover`] anchored to the input. Selecting an item dispatches `on_select` with the chosen value.

## Why you use it
- Keep UI behavior deterministic by composing from explicit nodes.
- Use it where behavior needs to stay testable and reviewable.
- Connect it through your existing action/reducer and environment flow.

## Minimal usage
```rust
use fission_widgets::Combobox;

let node = Combobox {
    ..Default::default()
}.into_node();
```

## Config surface
- `id`: `WidgetNodeId`
- `value`: `String`
- `items`: `Vec<String>`
- `is_open`: `bool`
- `width`: `Option<f32>`
- `max_popup_height`: `Option<f32>`
- `on_change`: `Option<ActionEnvelope>`
- `on_select`: `Option<Arc<dyn Fn(String) -> ActionEnvelope + Send + Sync>>`
- `on_toggle`: `Option<ActionEnvelope>`
