The Fission documentation site is now built with Fission's own static site target. The landing page and footer are ordinary Fission widgets, while the docs, reference, charts, cookbook, and blog pages are Markdown content routes rendered through the static site shell.

{/* truncate */}

This matters because the site is a practical example of the framework it documents. It uses custom widget routes, Markdown and MDX content, sidebars, generated table-of-contents links, light and dark themes, copied assets, favicons, client-side search, optional code highlighting, sitemap and robots output, structured metadata, and internal-link validation.

Use:

- `/docs` for guided learning and tutorials,
- `/reference` for framework, widget, chart, and command-line reference,
- `/docs/charts` for chart guides and the implemented chart catalog,
- `/docs/cookbook` for practical application walkthroughs.

The project lives in `documentation/`, and you can build it locally with `cargo fission site build --project-dir documentation` or serve it with `cargo fission site serve --project-dir documentation`.
