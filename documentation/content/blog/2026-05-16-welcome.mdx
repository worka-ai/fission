The Fission documentation site is a real Fission static site application.

The landing page, product pages, shared header, and footer are ordinary Fission widgets. The docs, reference, charts, cookbook, and blog pages are Markdown and MDX content routes rendered through the Fission static site shell. The final output is static HTML, CSS, JavaScript, images, metadata, search index files, sitemap data, and copied assets that can be deployed like any other static site.

{/* truncate */}

That matters because this site is not a separate documentation stack sitting beside the framework. It exercises the same model the docs describe: widget routes for designed pages, content routes for Markdown, explicit sidebars, generated table-of-contents links, light and dark themes, copied assets, favicons, client-side search, optional code highlighting, structured metadata, JSON-LD, sitemap and robots output, and internal-link validation.

If you are learning Fission, use the site in this order:

| Section | Use it for |
| --- | --- |
| [Docs](/docs/intro/) | Guided learning from setup through development, testing, packaging, and release. |
| [Cookbook](/docs/cookbook/build-a-counter/) | Copyable walkthroughs for common app tasks. |
| [Reference](/reference/overview/overview/) | Exact contracts for runtime APIs, widgets, charts, capabilities, platform targets, and CLI commands. |
| [Charts](/docs/charts/overview/) | Native chart families, catalog screenshots, data models, and interaction guidance. |

You can build the same site locally:

```bash
fission site check --project-dir documentation --release
fission site serve --project-dir documentation
fission site build --project-dir documentation --release
```

The project lives in `documentation/`. It is intentionally kept as a real-world example of how a production Fission static site is organized.
