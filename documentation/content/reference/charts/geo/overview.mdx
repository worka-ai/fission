# Geo

The Geo family contains 20 renderer-backed chart variants. Use this page as the entry point for choosing the right variant, then open the variant page for the screenshot, data shape, Rust API notes, and example code.

The chart family is still ordinary Fission UI. Each variant is constructed from typed Rust data, participates in layout and theme decisions, and can be driven by app state, reducers, resources, jobs, or services just like any other widget.

## Variants

| Chart | Data shape | Use when |
| --- | --- | --- |

| [Courier routes](./lines-courier-routes) | Point-to-point route lines. | Use it for logistics routing. |
| [Coverage score map](./map-coverage-score) | Coverage scores by region. | Use it for service coverage surfaces. |
| [Data center links](./geo-data-center-links) | Route-like links between points. | Use it for infrastructure network views. |
| [Dispatch routes](./geo-dispatch-routes) | Map regions plus route lines. | Use it when routes need region context. |
| [Expansion plan map](./map-expansion-plan) | Planning values by region. | Use it for geographic planning dashboards. |
| [Flight density lines](./lines-flight-density) | Dense route lines between points. | Use it for movement patterns. |

| [Incident severity map](./map-incident-severity) | Severity values mapped to region color. | Use it for incident command views. |
| [Market regions map](./map-market-regions) | Named regions with values. | Use it for regional comparisons where shape matters. |
| [Migration flow](./geo-migration-flow) | Region map plus movement routes. | Use it for source-to-destination stories. |

## Choosing within this family

Start with the user's question, not the visual effect. Pick the simplest variant that makes the answer clear, add interaction only when it reduces work for the user, and keep animation purposeful enough that it explains a change rather than decorating the page.
