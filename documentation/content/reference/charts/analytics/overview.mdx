# Analytics

The Analytics family contains 20 renderer-backed chart variants. Use this page as the entry point for choosing the right variant, then open the variant page for the screenshot, data shape, Rust API notes, and example code.

The chart family is still ordinary Fission UI. Each variant is constructed from typed Rust data, participates in layout and theme decisions, and can be driven by app state, reducers, resources, jobs, or services just like any other widget.

![Acquisition trend screenshot](/img/charts/analytics-acquisition-line.png)

## Variants

| Chart | Data shape | Use when |
| --- | --- | --- |

| [Analytics score gauge](./analytics-gauge-score) | One score value in a bounded range. | Use it for primary scorecards. |
| [Channel stack](./analytics-channel-stack) | Channel values stacked by category. | Use it for acquisition mix. |
| [Cohort heatmap](./analytics-cohort-heatmap) | Cohort matrix values. | Use it for retention and cohort intensity. |
| [Conversion funnel](./analytics-conversion-funnel) | Ordered conversion stage values. | Use it for product funnel analysis. |
| [Device mix donut](./analytics-device-donut) | Device values in a donut chart. | Use it when device share matters. |
| [Engagement calendar](./analytics-calendar-engagement) | Daily engagement values. | Use it for engagement consistency over time. |
| [Engagement routes](./analytics-route-engagement) | Map plus route lines. | Use it for spatial engagement flows. |
| [Feature treemap](./analytics-treemap-features) | Feature values in a hierarchy. | Use it for feature usage share. |
| [Journey graph](./analytics-journey-graph) | Journey nodes and links. | Use it for relationship-heavy behavior views. |
| [Journey sankey](./analytics-journey-sankey) | Journey stages and flow links. | Use it for user movement between stages. |
| [Market share pie](./analytics-market-pie) | Label/value market share pairs. | Use it for small whole comparisons. |
| [Portfolio sunburst](./analytics-sunburst-portfolio) | Portfolio hierarchy in radial layers. | Use it for hierarchical portfolio composition. |
| [Product radar](./analytics-radar-product) | Product dimensions as profile vectors. | Use it to compare product health dimensions. |

| [Source theme river](./analytics-source-river) | Source values over time. | Use it for changing source composition. |

## Choosing within this family

Start with the user's question, not the visual effect. Pick the simplest variant that makes the answer clear, add interaction only when it reduces work for the user, and keep animation purposeful enough that it explains a change rather than decorating the page.
