# Toolbox actions

![Toolbox actions screenshot](/img/charts/toolbox-actions.png)

Chart-level tool buttons for restore, save, zoom, and brush actions. Use it when a chart has explicit utility operations that should remain close to the data. The screenshot is captured from the native Fission chart gallery.

## What the chart is for

Toolbox actions belongs to the **Components and interaction** family. Its job is to make this data shape readable: `ChartInteraction toolbox action list.` In a production interface, choose this chart when the visual form makes the user's question faster to answer than a table or a simpler chart would.

Avoid it when the visual form makes the user estimate more than necessary; choose the simplest chart that answers the product question.

## Data model

`ChartInteraction toolbox action list.` Keep the data close to the type that describes it. Fission Charts is typed Rust, so a line uses `LineSeries`, a calendar heatmap uses `CalendarHeatmapSeries`, and chart components such as zoom, marks, and graphics are explicit fields on `Chart` rather than hidden string configuration.

## Rust API

| Field | Type | Notes |
| --- | --- | --- |
| `title` | `&str` | Names the chart for the screen, accessibility tree, and test output. |
| `interaction` | `ChartInteraction` | Defines the visible tool action list. |
| `width / height` | `f32` | Optional fixed size; omit them when the chart should flex inside Fission layout. |

## Example

```rust
use fission_charts::Chart;

let chart = Chart::new()
    .title("Toolbox actions")
    .width(720.0)
    .height(480.0);
```

## Interaction and animation

Charts can emit typed `ChartInteractionEvent` values when interaction is enabled. Handle those events in a reducer when the app needs hover, press, release, scroll, selection, or brush behavior. `ChartAnimation` stores duration, delay, stagger, easing, and reduced-motion behavior as deterministic chart data, so animation timing can be tested instead of being hidden in ad-hoc timers.

## Testing guidance

For this chart, test the data mapping first, then test lowering, then capture a live screenshot when visual output changes.
