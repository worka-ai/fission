# Microphone

The microphone capability asks the host for input-device availability, permission, and bounded audio capture.

## Public API

| Item | Value |
| --- | --- |
| Effect helper | `ctx.effects.microphone()` |
| Provider trait or host contract | `MicrophoneHost` |
| Test provider or test entrypoint | `MemoryMicrophoneHost` |
| Primary request types | `MicrophonePermissionRequest / MicrophoneCaptureRequest` |
| Primary success types | `MicrophoneAvailability / MicrophonePermission / MicrophoneCapture` |
| Error type | `MicrophoneError` |
| CLI value | `microphone` |

## Operations

| Method or operation | What it is for |
| --- | --- |
| `availability` | Emits or handles the `Microphone` host operation for this capability. |
| `request_permission` | Emits or handles the `Microphone` host operation for this capability. |
| `capture_audio` | Emits or handles the `Microphone` host operation for this capability. |
| `cancel_capture` | Emits or handles the `Microphone` host operation for this capability. |

## Platform configuration

Run `fission add-capability microphone --project-dir .`. Android receives `RECORD_AUDIO`. iOS receives `NSMicrophoneUsageDescription`. macOS packaged apps also need microphone usage text, and Windows packaged apps may need microphone capability metadata.

The CLI records the capability in `fission.toml` when the capability has a CLI value. Generated mobile configuration is written to the platform files that the host target already owns: Android uses `platforms/android/AndroidManifest.xml`; iOS uses `platforms/ios/Info.plist` and `platforms/ios/Entitlements.plist` when the feature needs entitlements. Desktop package metadata is target-specific and should be reviewed during packaging because Windows, macOS, and Linux expose different permission and distribution systems.

## Runtime behavior

A provider should return a typed unsupported error when the host cannot perform the operation. It should not silently succeed. That lets reducers show a useful message, hide unsupported controls, or offer another path without guessing from the platform name.
